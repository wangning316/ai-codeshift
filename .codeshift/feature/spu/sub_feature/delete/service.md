# 商品SPU_删除(Service)

## Service粒度
* 独立的商品SPU_删除Service类，ProductSpuDeleteService。

## 方法描述
* 删除商品SPU，同步删除关联的商品SKU

## 业务逻辑
1. 参数校验：根据传入的`商品SPU_删除(api)-请求参数（Query）-SPU编号`使用`商品SPU(mapper)_删除`的数据库操作对象查询数据库，校验待删除的SPU编号是否真实存在，若不存在则抛出`商品 SPU 不存在`业务异常，错误码：`1_008_005_000`。
2. 数据校验：根据查询到的商品SPU数据对象，校验SPU状态必须为`回收站`状态才可删除；若SPU状态不是`回收站`，则抛出`商品 SPU 不处于回收站状态`业务异常，错误码：`1_008_005_004`。
3. 数据库操作：使用`商品SPU_删除(mapper)`的数据库操作对象，根据`SPU编号`删除`商品SPU(entity)`。
4. 关联数据删除：使用`商品SPU_删除(mapper)`的数据库操作对象，根据`SPU编号`批量删除该商品下所有关联的SKU数据。
